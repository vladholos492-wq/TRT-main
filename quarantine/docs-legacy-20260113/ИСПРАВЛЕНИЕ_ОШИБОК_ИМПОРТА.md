# Исправление ошибок импорта в VS Code

## Проблема
VS Code показывает ошибки импорта (красные подчеркивания) для модулей:
- `telegram`, `telegram.ext`
- `dotenv`
- `aiohttp`, `aiofiles`
- `PIL` (Pillow)
- `pytesseract`
- и другие

## Решение

### Вариант 1: Использовать setup.bat (Рекомендуется)

1. **Откройте командную строку (CMD) или PowerShell**
2. **Перейдите в папку проекта:**
   ```
   cd C:\Users\User\Desktop\5656-main
   ```
3. **Запустите установку:**
   ```
   setup.bat
   ```
   Этот скрипт автоматически:
   - Проверит наличие Python
   - Установит все зависимости из `requirements.txt`
   - Создаст файл `.env` с настройками

### Вариант 2: Установка вручную через VS Code

1. **Откройте терминал в VS Code:**
   - Нажмите `Ctrl + ~` (тильда)
   - Или меню: `Terminal` → `New Terminal`

2. **Выберите интерпретатор Python:**
   - Нажмите `Ctrl + Shift + P`
   - Введите: `Python: Select Interpreter`
   - Выберите Python 3.8 или выше

3. **Установите зависимости:**
   В терминале VS Code выполните:
   ```
   python -m pip install -r requirements.txt
   ```
   
   Если не работает, попробуйте:
   ```
   python3 -m pip install -r requirements.txt
   ```
   или
   ```
   py -m pip install -r requirements.txt
   ```

### Вариант 3: Создать виртуальное окружение (Лучшая практика)

1. **Создайте виртуальное окружение:**
   ```
   python -m venv venv
   ```

2. **Активируйте его:**
   - В PowerShell:
     ```
     .\venv\Scripts\Activate.ps1
     ```
   - В CMD:
     ```
     venv\Scripts\activate.bat
     ```

3. **Установите зависимости:**
   ```
   pip install -r requirements.txt
   ```

4. **Выберите интерпретатор в VS Code:**
   - `Ctrl + Shift + P`
   - `Python: Select Interpreter`
   - Выберите `.\venv\Scripts\python.exe`

## Проверка установки

После установки проверьте, что все работает:

```python
python -c "import telegram; print('OK')"
python -c "import dotenv; print('OK')"
python -c "import aiohttp; print('OK')"
```

Если все команды выводят "OK", значит зависимости установлены правильно.

## Если ошибки остаются

1. **Перезапустите VS Code** после установки зависимостей
2. **Проверьте, что выбран правильный интерпретатор Python** в VS Code
3. **Убедитесь, что Python добавлен в PATH** системы
4. **Проверьте версию Python:** должна быть 3.8 или выше
   ```
   python --version
   ```

## Дополнительная информация

- Если Python не найден в терминале, но установлен:
  - Перезагрузите компьютер после установки Python
  - Или добавьте Python в PATH вручную:
    - Откройте "Переменные среды" в Windows
    - Добавьте путь к Python в переменную PATH
